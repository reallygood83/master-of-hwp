- 날짜: 2026-04-19

## 목표
- cargo 없이 Node + WASM 경로로 HWPX 텍스트 추출이 가능한지 확인
- Python MCP 어댑터가 브리지를 호출해 동일한 결과를 돌려받는지 확인
- 회귀를 잡기 위한 pytest 스위트를 추가하고 초록불(green)로 만들 것

## 실행 방법
1. `cd mcp-server/bridges && npm install`
2. `node smoke_test.mjs` (브리지 단독 왕복)
3. Node 스크립트로 `/Users/moon/Desktop/master-of-hwp/tmp_verify/roundtrip.hwpx` 생성 (createEmpty → createBlankDocument → insertText → exportHwpx)
4. Python에서 `RHWPAdapter().extract_text(...)` / `extract_structure(...)` 호출
5. `.venv/bin/pytest -q tests/`

## 기대 결과
- `smoke_test.mjs`가 삽입한 텍스트(`안녕 Hello 123`)를 그대로 복원
- Python 어댑터가 한글 문자열(`한글 테스트 문단 하나`, `pytest 한글 roundtrip 123`)을 손실 없이 추출
- pytest 모두 통과

## 실제 결과
- `node smoke_test.mjs` 출력:
  ```json
  {
    "ok": true,
    "sections": 1,
    "paragraphs": 1,
    "chars": 12,
    "sample": "안녕 Hello 123"
  }
  ```
- Python 어댑터 왕복:
  - `source_format=hwpx`, `section_count=1`, `paragraph_count=1`
  - `text='한글 테스트 문단 하나'` (char_count=12)
  - `extract_structure`가 1개의 section / paragraph를 그대로 반환
- pytest: `7 passed in 0.23s` (txt/md/hwpx 본선, 권한 검사, 미지원 포맷, 구조 추출 포함)

## 판정
- 성공: 브리지 경로가 실제 동작하며 Python 쪽 `_extract_with_rhwp`가 의도대로 동작함
- 이유: 동일 텍스트가 생성→저장→재추출 단계에서 바이트 단위로 보존되었고 pytest 초록불

## 이슈
- 시스템 기본 `python3`(3.9)에는 `fastmcp>=3.2` 휠이 없어 `.venv`는 Python 3.13 필요

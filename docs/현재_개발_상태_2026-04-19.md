# 현재 개발 상태 - 2026-04-19

## 핵심 상태
`master-of-hwp`는 이제 **MCP 기반 문서 에디터 코어**, **실제 rhwp bridge 기반 HWP/HWPX 읽기 루트**, 그리고 **두 단계의 HWP/HWPX 저장 루트**를 확보했다.

## 실제 동작 확인된 것
- TXT/MD 문서 열기, 구조 추출, 문단 수정, 문단 삽입, 저장, 저장 검증
- `.hwp` 실제 텍스트 추출 성공
- `.hwp` 실제 구조 추출 성공
- `.hwpx` 실제 텍스트 추출 성공
- `.hwpx` 실제 구조 추출 성공
- `.hwpx`에서 실제 table metadata 추출 성공 (`getPageControlLayout` 기반)
- 로컬 bridge 자동 감지 및 기본 extractor로 설정
- 수정된 세션 텍스트를 `.hwp`로 저장 후 재검증 성공
- 수정된 세션 텍스트를 `.hwpx`로 저장 후 재검증 성공
- 기존 `.hwpx` 문서를 로드한 뒤 문단 치환/삽입을 재적용하고 다시 `.hwpx`로 저장 후 재검증 성공
- 기존 `.hwp` 문서를 로드한 뒤 문단 치환/삽입을 재적용하고 다시 `.hwp`로 저장 후 재검증 성공
- `python3 -m pytest -q` 기준 14개 테스트 전부 통과

## 저장 관련 현재 진실
현재 저장은 두 단계로 구분해야 한다.

### 1. blank-document text export
- 세션 최종 텍스트를 새 blank document에 삽입
- `exportHwp()` / `exportHwpx()`로 저장
- 텍스트 내용 저장은 가능하지만 원본 구조를 보존하지 않음

### 2. roundtrip operation replay
- 기존 원본 `.hwp/.hwpx`를 다시 로드
- 기록된 문단 치환/삽입 연산을 재적용
- `exportHwp()` / `exportHwpx()`로 저장

이 방식은 blank export보다 원본 구조를 더 많이 유지한다.

## 저장의 현재 한계
아직 완전한 구조 보존 저장은 아니다.
현재 roundtrip save는 아래 수준이다.

- ✅ 유효한 `.hwp` / `.hwpx` 파일 생성
- ✅ 수정한 텍스트 내용을 실제 HWP/HWPX로 저장
- ✅ 일부 원본 구조를 유지한 roundtrip save 시작
- ✅ 실제 HWPX 파일에서 table metadata 추출
- ❌ table creation roundtrip persistence는 아직 검증 실패
- ❌ 표/개체/서식/레이아웃의 완전 보존 보장
- ❌ 모든 편집 연산을 정밀 재현하는 full roundtrip editor

## 아직 남은 것
